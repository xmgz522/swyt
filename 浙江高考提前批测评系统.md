# 浙江高考提前批志愿填报系统

## 1. 产品定位

面向浙江高考家长和考生的**提前批志愿填报数据查询与智能匹配系统**。

核心理念：**用数据解决问题，简单高效**。

不做花里胡哨的功能堆砌，只做三件事：
1. **院校数据** — 提前批院校的完整招生信息
2. **专业数据** — 专业介绍、开设院校、就业方向
3. **业务数据** — 三年录取数据联查、特殊类型招生数据

加上**登录 + 个人成绩匹配**，让家长打开就能查、查完就有结论。

---

## 2. 核心闭环

```text
家长/考生进入系统
  → 登录，填写学考成绩（几A几B）
  → 首页看到个性化匹配结果
  → 浏览院校库，查看三年录取数据
  → 浏览专业库，了解专业详情
  → 三年数据联查，对比分析
  → 得出填报方案
```

**没有模拟考试，没有刷题，没有错题本。** 就是数据查询 + 智能匹配。

## 3. 用户角色

### 3.1 家长/考生

使用微信小程序或 PC 端。

主要操作：
- 登录注册
- 填写学考等级（几A几B几C）
- 查看匹配院校推荐
- 浏览院校库、查看院校详情和三年录取数据
- 浏览专业库、查看专业详情
- 三年数据联查（跨院校、跨年份对比）
- 查看特殊类型招生信息

### 3.2 管理员

使用管理后台。

主要操作：
- 管理院校数据（含三年录取数据）
- 管理专业数据
- 管理特殊类型招生数据
- 管理推荐匹配规则
- 管理用户/注册码
- 批量导入数据

## 4. 三大核心数据模块

### 4.1 院校数据

#### 院校基础信息
- 学校名称
- 所在省份/城市
- 学校层次（985/211/双一流/普通本科）
- 学校类型（综合/理工/师范/财经/医药等）
- 学校简介
- 官网链接
- 学校 Logo

#### 提前批招生信息
- 招生类型（三位一体/提前批/强基计划等）
- 招生计划人数
- 报名条件
- 学考等级要求（如：A≥3, B≥2）
- 综合素质评价要求
- 考核方式（笔试/面试/笔试+面试）
- 考核内容说明
- 报名时间
- 考核时间

#### 三年录取数据（核心亮点）

每所院校存储近三年（如 2024/2023/2022）的录取数据：

| 字段 | 说明 |
|------|------|
| 年份 | 2024 / 2023 / 2022 |
| 招生计划数 | 该年招生人数 |
| 报名人数 | 该年报名人数 |
| 通过初审人数 | 初审通过人数 |
| 入围比例 | 通过率 |
| 录取人数 | 最终录取人数 |
| 录取最低分 | 综合成绩最低分 |
| 录取平均分 | 综合成绩平均分 |
| 录取最高分 | 综合成绩最高分 |
| 高考分数线 | 高考成绩要求 |
| 学考等级要求 | 该年的学考门槛 |
| 综合成绩折算方式 | 如：高考60%+校测30%+学考10% |

### 4.2 专业数据

#### 专业基础信息
- 专业名称
- 专业代码
- 所属学科门类（哲学/经济学/法学/教育学/文学/理学/工学/医学等）
- 学制（4年/5年）
- 授予学位
- 专业简介
- 核心课程
- 就业方向
- 薪资参考
- 适合人群

#### 专业与院校关联
- 哪些院校开设了该专业
- 各院校该专业的提前批招生计划
- 各院校该专业的历年录取分数

### 4.3 业务数据（特殊类型招生）

#### 招生类型
- 三位一体
- 强基计划
- 高校专项计划
- 地方专项计划
- 艺术类提前批
- 体育类提前批
- 军校/公安/消防
- 定向招生
- 其他特殊类型

#### 每种类型的数据
- 适用院校列表
- 报名条件
- 时间安排
- 录取规则
- 三年录取数据
- 注意事项

## 5. 用户端功能

### 5.1 登录与个人信息

- 手机号 + 密码登录
- 注册需要注册码
- 填写个人信息：
  - 姓名
  - 学考等级（A/B/C/D/E 各几个）
  - 高考预估分（选填）
  - 意向地区（选填）
  - 意向专业方向（选填）

### 5.2 首页 — 个性化匹配

登录后首页直接展示：

- **我的匹配结果**：基于学考成绩的院校推荐
  - 冲刺院校（有机会但竞争大）
  - 稳妥院校（条件匹配度高）
  - 保底院校（大概率能上）
- **快捷入口**：院校库、专业库、数据联查
- **最新动态**：招生政策变化提醒

### 5.3 院校库

- 院校列表（支持搜索、筛选）
  - 按地区筛选
  - 按学校层次筛选
  - 按招生类型筛选
  - 按学考要求筛选（如：我是3A2B能报哪些）
- 院校详情页
  - 学校基本信息
  - 提前批招生信息
  - **三年录取数据对比表**（核心）
  - 开设专业列表
  - 报考建议

### 5.4 专业库

- 专业列表（支持搜索、按学科门类筛选）
- 专业详情页
  - 专业介绍、核心课程、就业方向
  - 开设该专业的提前批院校列表
  - 各院校该专业录取数据

### 5.5 三年数据联查（核心亮点）

一个综合查询页面，家长可以：

- **按院校查**：选一所院校，看它三年的招生/录取数据变化趋势
- **按专业查**：选一个专业，看各院校该专业的三年数据对比
- **按条件查**：输入自己的学考等级，一键筛选出所有符合条件的院校和专业
- **多校对比**：选择 2-4 所院校，横向对比三年录取数据

展示形式：
- 数据表格（可排序）
- 趋势图（折线图展示三年变化）
- 对比图（柱状图横向对比）

### 5.6 特殊类型招生

- 按招生类型分类浏览
- 查看各类型的报名条件、时间、流程
- 查看适用院校
- 查看三年录取数据

## 7. 数据模型

### 7.1 核心实体

| 实体 | 说明 |
|------|------|
| users | 后台管理员 |
| students | 家长/考生用户 |
| schools | 院校基础信息 |
| school_admissions | 院校招生信息（按年份） |
| school_enrollment_data | 院校三年录取数据（核心） |
| majors | 专业信息 |
| school_majors | 院校-专业关联（含该专业在该校的录取数据） |
| special_admissions | 特殊类型招生信息 |
| recommend_rules | 推荐匹配规则 |
| invite_codes | 注册码 |

### 7.2 关键表结构

#### school_enrollment_data（三年录取数据）

```sql
CREATE TABLE school_enrollment_data (
  id INT PRIMARY KEY AUTO_INCREMENT,
  school_id INT NOT NULL,
  year INT NOT NULL,                    -- 年份：2024/2023/2022
  admission_type VARCHAR(50),           -- 招生类型：三位一体/强基等
  plan_count INT,                       -- 招生计划数
  apply_count INT,                      -- 报名人数
  pass_count INT,                       -- 通过初审人数
  pass_rate DECIMAL(5,2),               -- 通过率
  enroll_count INT,                     -- 录取人数
  min_score DECIMAL(5,1),               -- 最低分
  avg_score DECIMAL(5,1),               -- 平均分
  max_score DECIMAL(5,1),               -- 最高分
  gaokao_min_score DECIMAL(5,1),        -- 高考最低分
  xuekao_requirement VARCHAR(100),      -- 学考要求
  score_formula VARCHAR(200),           -- 综合成绩折算方式
  remark TEXT,                          -- 备注
  created_at TIMESTAMP,
  updated_at TIMESTAMP,
  UNIQUE KEY (school_id, year, admission_type)
);
```

### 7.3 预留字段

所有业务表预留 `app_id`，默认 `default`，为后续多租户做准备。

## 11. 验收标准

系统完成后，应该能演示以下流程：

1. 家长打开系统，注册登录
2. 填写孩子的学考成绩（如 3A4B3C）
3. 首页立刻显示匹配的冲刺/稳妥/保底院校
4. 点进一所院校，看到完整的三年录取数据
5. 打开专业库，查某个专业在哪些院校有提前批招生
6. 打开三年联查，选两所院校横向对比
7. 看到清晰的数据表格和趋势图
8. 家长心里有数，知道该报哪几所
