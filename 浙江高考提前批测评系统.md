# 浙江高考提前批测评系统

## 1. 产品定位

浙江高考提前批测评系统是整体白标系统底座的第一个落地版本。

它面向浙江高中生、升学规划机构、培训机构和相关招生服务机构，提供：

- 学生信息维护
- 几 A 几 B 测评
- 院校智能推荐
- 院校详情展示
- 模拟卷练习
- 后台批卷
- 薄弱点分析
- 测评报告生成

第一版目标不是做大而全，而是先完成一个能演示、能销售、能交付的业务闭环。

---

## 2. 第一版核心闭环

MVP 核心闭环：

```text
学生进入小程序
  -> 填写个人信息和几A几B
  -> 系统推荐可报院校
  -> 学生查看院校详情
  -> 学生完成模拟卷
  -> 老师后台批改主观题
  -> 系统生成成绩和薄弱点分析
  -> 学生查看测评报告
  -> 机构老师后台跟进转化
```

- 风险提示

- 学校地区
- 学校类型
- 学校简介

- 学考等级要求
- 面试形式
- 往年录取情况

- 报考建议
- 关联模拟卷

- 模拟卷 1
- 模拟卷 2

- 适合人群
- 报考建议

### 5.3 推荐规则管理

每个院校可以配置推荐规则。

第一版规则条件：

- A 等第数量 大于等于
- B 等第数量 大于等于
- C 等第数量 小于等于
- 意向地区 包含
- 意向专业方向 包含

推荐结果类型：

- 冲刺
- 稳妥
- 保底
- 不推荐

示例：

```text
A等第数量 >= 3 AND B等第数量 >= 2 -> 稳妥
A等第数量 >= 2 AND B等第数量 >= 2 -> 冲刺
A等第数量 >= 4 AND B等第数量 >= 1 -> 保底
```

## 11. 第一版验收标准

第一版完成后，应该可以演示以下流程：

1. 超管配置系统名称、Logo、主题色。
2. 后台录入几所提前批院校。
3. 后台为院校配置推荐规则。
4. 学生进入小程序填写几 A 几 B。
5. 系统自动推荐冲刺、稳妥、保底院校。
6. 学生点击查看院校详情。
7. 后台创建一套模拟卷。
8. 学生完成模拟卷。
9. 客观题自动批改。
10. 老师后台批改主观题。
11. 学生查看成绩和报告。
12. 老师后台查看学生线索和报告。

---

## 12. 一句话总结

浙江高考提前批测评系统是第一个可销售版本，第一版优先完成“学生填信息 -> 推荐院校 -> 查看详情 -> 做模拟卷 -> 后台批改 -> 生成报告”的闭环，同时底层保持可配置，为后续换皮成高职、考研、高考志愿等系统做准备。
